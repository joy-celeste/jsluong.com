Introduction

Hey! So you want to make readers, like this one (opens in new tab).

Readers are made up of four parts: the table of contents, the overview, slides/transcripts, and readings. These parts correspond to the various sections under the table of contents:

Generally, reader creation goes like this. Go ahead and click to skip to a particular section:

  1. Format transcripts and collect readings
  2. Create the overview
  3. Put overview, transcripts, readings together in Adobe Reader
  4. Create table of contents
  5. Compress file size and upload used resources
Why this order? You need to know what's in the readers (transcripts, readings) to make the overview. To put the reader together, you need the overview/transcripts/readings. And then to create the table of contents, you need to have everything else prepared (how else would you know what the page numbers are going to be?).

Things Needed to Make Readers

Before getting started, it's important to make sure you have all the materials you need to generate the reader. You will need this information to create the overview anyway, so best have everything here before getting started.
  1. learning outcomes
    • to provide on the overview
  2. list of readings
    • both required and optional; though I don't typically include the optional readings themselves, I provide links to them in the overview. Also sometimes some readings may not be incuded for you, so you have to go hunt it down on the Internet.
  3. the readings themselves
    • I mean the files--usually provided in .pdfs or Word docs. Occasionally only a site will be provided; see the "Collect Readings" section to see how I deal with this
  4. link to lectures themselves
    • to provide on the overview
  5. link to lecture mp3s
    • to provide on the overview/li>
  6. lecture transcripts
    • The text itself--to create the transcripts and to provide on the overview. Generally you get this in a .txt or a Word doc file.
  7. discussion prompts
    • optional--some courses want to include these; ask whoever gave you this assignment if they want to include them

Formatting Transcripts

Formatting the text and slides such that they look like this:

For this section, you need:

  1. Transcript text file
  2. Slides
  3. Microsoft Word
  4. Microsoft PowerPoint
  5. A template just to keep things standardized across readers--I use this one.
Steps:
  1. Open up the transcript template.
  2. Edit the header to match the name and video length of the lecture. Include the name of the lecturer if you wish (nowadays I just leave it out--that information isn't really important). Edit the [Watch Online] link to go to the lecture video. Usually the link looks something like https://d2egua14l5soou.cloudfront.net/MPHW205_v13/indexes/w205_v13_l9_1.html.

  3. Now, sometimes when you format transcripts you'll get a word doc with all the slides and text already in there. Nice! All you have to do is copy and paste everything into the transcript template and move on to step 8. Sometimes you don't, though, so you need to insert in the PowerPoint slides yourself.
  4. There are two ways to get slides.

    a) If you already have the .ppt file, open the file in PowerPoint. File > Save As > select PNG Portable Network Graphics Format under Save as type. This lets you export all the slides as PNG, the highest quality image type available.

    b) At the prompt, select Every Slide...

    c) ...and you'll have all the slides saved as a folder wherever you chose to save them! Yay!

  5. If you don't have the slides, you'll have to take screenshots from the lecture video itself to produce them. This is how I do that (if you have another way, feel free to use that method):

    aa) Open up the lecture in a browser:

    bb) View Source (Ctrl+U), look for the link that looks like "http:// . . . .mp4"

    cc) Open link in new tab, save video by (Ctrl+S).

    dd) Open video in VLC. Use the command (Shift+S) to save screenshots to your Pictures folder (when you save an image it tells you where VLC saved it to).

  6. Now you have to insert the slide images into their rightful place. So you need to play the video to know where to put those slide images. I like to download the .mp4 and play it in VLC (steps aa through cc above) because when you play the video directly from the online site, it's really laggy and you always have to wait a few seconds for the video to buffer if you're going back a few seconds in the video. Also, in VLC you're allowed to change the speed of playback. :)
  7. Now you have something like this!

  8. Almost done! Make sure the font is Calibri size 12, and select the entire document (Ctrl+A) and right click -> change line spacing to Multiple at 1.15. This is the font/line space option I've been using for a while, so just to keep everything on the same page... though you're free to change this if you want. Just keep it consistent across a course-worth of readers! :)
  9. If you're copy and pasting from an old set of transcripts, above some of the slides might be something like "Slide 1" or "Slide 2" to show the slide number. Go ahead and delete those lines if you see them. Image of what I'm talking about:
  10. When you're done, select all the files in the folder, right click -> convert to Adobe PDF. Alternatively, open the file in Word and go to File > Save Save as Adobe PDF.

    11. You're done! Your final .pdf should look like this: click here to see.

Collecting Readings

The goal to is have as much required readings as possible as .pdfs to put into the reader (I don't include optional readings in the reader; instead I just provide a link to them so if students want to check them out they can download them).

Usually the instructors provide you with all the readings in you need. Sometimes they forget to include something, though. Generally I try Googling for the required file, and most of the time I can find it so it's alright. Sometimes I really do have to go back and ask for it, though. It happens.

Sometimes instructors provide a link to a website to read some text (online newspaper article)--in those cases, I like to save the page as a PDF so people don't need internet to access those readings. For example, with a site of like this, I would make a PDF out of. So hunt for a little printer icon (if possible) or just Ctrl+P and print the webpage as a PDF. Done!

If a reading spans multiple pages I would copy and paste everything into a word doc and save the word doc as a pdf.


Create Overview

So here's an example of an overview (still as a word doc). Go ahead and open it up and use that as a template. I pulled all of this information for this particular week's overview from here (I've invited you, Vanessa, to this course site).

Create one document per week. Frequently under each week there are separate "Modules" or "Topics"--for each of these, I separate them into different pages. This is so when we add the readings and transcripts to make the reader, each module/topic has its own overview page.

Again, to make the overview, generally I include an Introduction to the Week, the learning outcomes, the lecture (link to lecture and length of the video, plus mp3 and transcript link if it's available), list of readings (for the required readings, I put an asterisk next to the ones that will be included in the reader; for the optional readings, I provide a link to them). Sometimes they'll ask to skip the introduction or include a section (in this case, the discussion) to the week. Genearlly if they don't say anything, though, I'll go with the above things.

Yep... so just change the links and text. Make sure the font stays as 12 pt Calibri!

When you're done, save each Week Overview as a .pdf (like you saved the transcripts as a .pdf).


Putting It All Together in Adobe Reader

  1. Open your .pdf'd overview in Adobe Reader.

  2. While opened on the first overview page, insert the first lecture with Insert From File on the right.

  3. The first page of the first lecture should be selected (in your viewing window). On top of the left panel, click on the New Bookmark button. Type in "Slides and Transcripts". When you click on "New Bookmark", it creates a bookmark on that page. In the end, the plug-in will take these bookmarks and create the Table of Contents with them.

  4. Make another bookmark on the first page of the lecture--this time, type the actual name (and length) of the lecture.

  5. Repeat step 4 until you have all the required lectures. Then repeat step 3, this time creating a section for "Required Readings". Then repeat step 4 again for the readings. Move all the bookmarks to the right hierarchy like so:

  6. FYI: Sometimes when you add readings, they add extra bookmarks like so. Just delete those bookmarks (Kaplan_The use of logic models.pdf in this case).

Okay, you put everything together in Adobe Reader! Now to make the Table of Contents.

Creating the Table of Contents, Renumbering Pages, Add ToC Image.

  1. Plug-Ins > Table of Contents > Craete TOC from Bookmarks

  2. You get this window. Please import these settings with this file. Import with the "Load Settings..." button. Change the "Title" to "PH WXXX: Course Name - Week X Reader" according to the name of the course/course number/week of the reader you're making.

  3. Hit OK.

  4. Select all the pages from the left panel, and click "Number Pages..." We're renumbering all the pages to start from 1, just in case adding the new ToC didn't change the page numbers.

  5. Use these settings, hit OK.

  6. Now, add the Table of Contents header image. "Add Images" at the right... Add this image.

  7. Align image. Use your keyboard up/down/left/right buttons to move it pixel by pixel. I just eyeball it until it seems right.


Footer (Page numbers, "Return to Table of Contents" Button)

  1. You now have a file that has the table of contents and reader pages put together, but without page numbers and a button at the bottom that says "Return to Table of Contents." Before doing anything else, save a copy of this file and put it somewhere safe. I name this copy "Week X (without page numbers or return to table of contents button)". It's important to save this because sometimes you need to add more pages or delete pages from a reader at an instructor's request; and it's impossible to remove the page numbers and table of contents button once you put them on the reader! So make a "clean" copy and put it aside incase you need to make revisions to the reader. It's good to have a backup incase Adobe crashes in the next couple of steps (it has happened before).
  2. After having made a copy, click the "Header & Footer" button at the right panel. Click "Add New". ("Replace Exisiting" will remove all the headers from the transcripts and overviews).

  3. Use these settings.

  4. Under "Page Range Options", go from page 2 to whatever the last page of the reader is. Hit OK.

  5. Hit OK again and the page numbers should be added.
  6. While you're on Page 2, hit the Add Interactive button under Interactive Objects. Place it anywhere.

  7. Hit All Properties.

  8. Use these properties.

    For this one: Make sure "Select Trigger" is "Mouse Down" and "Select Action" is "Go to a page view". Click "Add", scroll to page 1, and then confirm. Then click "Go to a page view" and "Edit", and change the zoom to "Inherit" (so people's zoom setting doesn't change when they click on the button).

  9. Right click the newly created button, and Duplicate Across Pages.

  10. Go from page 2 to the last page of the document.

  11. Done! Save it all.
